🔧 Naprawa błędu: auth/api-key-not-valid

❌ Problem: Błąd Firebase: Error (auth/api-key-not-valid. Please pass a valid API key.)
📋 Przyczyna: API Key w aplikacji nie jest prawidłowy lub projekt Firebase został zmieniony/usunięty.

🔍 Możliwe przyczyny

1. API Key został zmieniony w Firebase Console

Firebase Console mógł wygenerować nowy API Key lub stary został usunięty.

2. Projekt Firebase został usunięty lub przeniesiony

Projekt aplikacje-3068f może nie istnieć lub została zmieniona jego konfiguracja.

3. API Key ma ograniczenia

Klucz może mieć ograniczenia IP, domenowe lub usługowe które blokują dostęp z localhost.

✅ Rozwiązanie krok po krok

Krok 1: Sprawdź Firebase Console

  1. Otwórz 🔗 Firebase Console
  2. Sprawdź czy projekt aplikacje-3068f istnieje na liście projektów
  3. Jeśli nie ma projektu, możliwe że został usunięty lub przeniesiony

Krok 2: Pobierz nową konfigurację

Jeśli projekt istnieje:

  1. Kliknij na projekt aplikacje-3068f
  2. Przejdź do Project Settings (ikona koła zębatego)
  3. Znajdź sekcję "Your apps"
  4. Jeśli istnieje aplikacja webowa, kliknij na nią
  5. Skopiuj nową konfigurację Firebase

Krok 3: Utwórz nową aplikację (jeśli nie istnieje)

Jeśli nie ma aplikacji webowej:

  1. W Project Settings kliknij "Add app"
  2. Wybierz ikonę web </>
  3. Nazwij aplikację np. "RekruTracker Web"
  4. Włącz Firebase Hosting (opcjonalnie)
  5. Skopiuj wygenerowaną konfigurację

Krok 4: Zaktualizuj konfigurację w aplikacji

Zastąp starą konfigurację w plikach:

📁 Pliki do aktualizacji:
• firebase-init.js
• add-application.js
• login.html
• wszystkie pliki testowe

Nowa konfiguracja powinna wyglądać tak:

const firebaseConfig = {
  apiKey: "NOWY_API_KEY",
  authDomain: "aplikacje-3068f.firebaseapp.com",
  projectId: "aplikacje-3068f",
  storageBucket: "aplikacje-3068f.appspot.com",
  messagingSenderId: "869193309880",
  appId: "NOWY_APP_ID"
};

Krok 5: Sprawdź ograniczenia API Key

  1. W Firebase Console przejdź do Project Settings
  2. Sprawdź sekcję "Web API Key"
  3. Kliknij "Restrict key" lub "Manage key in Google Cloud Console"
  4. Sprawdź czy nie ma ograniczeń IP lub domenowych
  5. Upewnij się, że localhost jest dozwolony

🚀 Alternatywne rozwiązanie - nowy projekt

⚠️ Jeśli projekt został usunięty: Możesz utworzyć nowy projekt Firebase i przenieść dane.

Utworzenie nowego projektu

  1. W Firebase Console kliknij "Create a project"
  2. Nazwij projekt np. "rekrutracker-new"
  3. Włącz Google Analytics (opcjonalnie)
  4. Dodaj aplikację webową
  5. Skonfiguruj Firestore i Authentication
  6. Zaktualizuj konfigurację w aplikacji

🧪 Testy po naprawie

Po aktualizacji konfiguracji przetestuj:

🧪 Test API Key 🔐 Test logowania 📱 Test Google Auth

📞 Jeśli nadal nie działa

💡 Dodatkowe kroki:
  • Wyczyść cache przeglądarki (Ctrl+Shift+Delete)
  • Sprawdź konsolę przeglądarki (F12) pod kątem dodatkowych błędów
  • Upewnij się, że Authentication i Firestore są włączone w Firebase Console
  • Sprawdź czy Google Provider jest aktywny w Authentication > Sign-in method
